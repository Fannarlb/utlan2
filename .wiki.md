# Project Summary
The project is an internal dealership car loan tracker that facilitates the management of car loans within a dealership. It allows users to select a salesman, choose a car by license plate, and enter customer details to create a loan record. The application features a mobile-friendly interface for dealership staff to efficiently track active loans and search loan history. Recent updates have added new fields for customer identification and notes, improved the user interface with dark mode, and enhanced loan management functionalities.

# Project Module Description
- **Loan Management**: Create, view, and manage car loans, including checking out cars and marking them as returned.
- **Salesman and Car Selection**: Users can select a salesman and car, with alphabetical sorting of salesmen and checks to prevent the checkout of cars already on loan.
- **Customer Information**: Input fields for customer name, phone number, and now includes notes and customer identification (Kennitala) for enhanced tracking.
- **Active Loans Display**: A dedicated page to view all currently active loans with relevant details.
- **Loan History**: A searchable interface for reviewing past loans.

# File Description Inventory
- **backend/**: Contains backend logic, including models, services, and routes.
- **frontend/**: Contains the React-based frontend application, including UI components and pages for user interaction.

# Technology Stack
- **Backend**: Python, FastAPI, SQLAlchemy, Alembic for database migrations.
- **Frontend**: React, TypeScript, Vite, Tailwind CSS for styling.
- **Database**: SQL-based database to manage loans, cars, and salesmen.

# Usage
1. **Install Dependencies**:
   - For the backend: Navigate to the `app/backend` directory and run the command to install required packages.
   - For the frontend: Navigate to the `app/frontend` directory and run the command to install required packages.

2. **Build the Application**:
   - Run the build command for both backend and frontend to compile the application.

3. **Run the Application**:
   - Start the backend server and the frontend development server using the respective commands.
